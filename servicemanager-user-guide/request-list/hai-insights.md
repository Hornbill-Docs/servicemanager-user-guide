# HAi Insights

HAi Insights provides a quick view of which requests are the most important to be worked on.

![HAi Insights form on the request list](/_books/servicemanager-user-guide/request-list/images/hai-insights-request-list.png)

## Before you begin

To use HAi Insights with the request list, you need the following:

* **Register for the closed beta program**: HAi features are currently in a closed beta. You can access these features after you register. Contact your Hornbill customer success representative to join the program.
* [HAi Insights Role](/esp-config//esp-config/organizational-data/roles#hai-roles): Your user account must have this platform role.
* [HAi Request User](/servicemanager-config/setup/service-manager-roles#hai-roles): Your user account must have this Service Manager role.

## Settings

You can adjust how the information is presented to you in HAi Insights. Open HAi Insights from the right panel and select the settings cog in the top right.

![HAi Insights Settings form](/_books/servicemanager-user-guide/request-list/images/hai-insights-settings.png)

* **Request types to prioritize**: Select which request types you want displayed in HAi Insights.
* **Number of requests to prioritize**: Set the maximum number of requests that will be displayed in the Prioritized Requests list. This setting has a minimum of 10 and a maximum of 25.
* **Source request ordering**: Change the order in which requests with the same prioritization will be displayed.
* **Anomaly events to show**: Select between new and active anomaly events to be displayed.
* **Maximum detection events to display**: Select between 1 and 20 for the maximum number to display.
